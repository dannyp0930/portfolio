# .impeccable.md — Design Context for SH Portfolio

## Design Context

### Users
채용 담당자, 인사팀, 동료 개발자 등 포트폴리오를 평가하는 전문가가 주 방문자. 짧은 시간 안에 개발자의 역량과 신뢰감을 판단하는 맥락에서 방문하므로, 첫인상과 정보의 명확한 전달이 핵심. 모바일/데스크탑 양쪽에서 쾌적하게 탐색 가능해야 함.

### Brand Personality
**차분한 · 신뢰감 있는 · 현대적인**

군더더기 없이 깔끔하되, 차갑지 않은 온기가 있다. 과장 없이 실력으로 말하는 개발자의 인상. Toss처럼 한국 사용자에게 자연스러운 타이포그래피와 여백, 절제된 모션으로 전문성을 드러낸다.

### Aesthetic Direction
- **레퍼런스**: Toss — 강한 정보 위계, 넉넉한 여백, 의도 있는 마이크로인터랙션, 한국어 타이포그래피의 정석
- **현재 팔레트**: Peach Fuzz(`#ffdab9`) + 딥 버건디(`#522a28`) — Pantone 영감. 추후 어드민에서 커스터마이징 가능하도록 설계 예정
- **Anti-reference**: 화려한 파티클 효과, 무거운 배경 애니메이션, 정보보다 장식이 앞서는 레이아웃
- **모드**: 라이트/다크 모드 모두 지원 (next-themes 기반)
- **폰트**: KBO Dia Gothic (본문 · 섹션 타이틀), Parisienne (브랜드 포인트용 장식체)

### Design Principles

1. **신뢰 먼저 (Trust by Default)**
   레이아웃은 즉시 역량을 전달해야 한다. 명확한 타이포그래피 위계, 충분한 여백, 정보의 밀도를 절대 장식보다 우선시한다.

2. **색상은 변수다 (Colors Are Variables)**
   모든 색상은 CSS 변수 또는 Tailwind 토큰(`theme`, `theme-sub`)에 기반한다. 컴포넌트에 hex 하드코딩 금지. 어드민 커스터마이징을 전제로 설계한다.

3. **모션은 의미가 있을 때만 (Motion with Purpose)**
   스크롤 reveal, hover lift, 전환 등 모든 애니메이션은 콘텐츠의 흐름을 돕거나 인터랙션 피드백을 제공하는 경우에만 사용. `prefers-reduced-motion` 항상 존중.

4. **한국어 타이포그래피 우선 (Korean-first Typography)**
   KBO Gothic은 한글과 영문 모두 자연스럽게 소화한다. 줄바꿈, 자간, 행간은 한국어 가독성 기준으로 조정. 불필요한 `break-all` 대신 `break-keep` 선호.

5. **정보 밀도, 단 숨통은 남겨라 (Dense but Breathable)**
   섹션 간 여백은 넉넉하게, 섹션 내부는 관련 정보를 묶어 그룹핑. 스크롤 한 번에 맥락이 잡혀야 하고, 읽다가 지쳐서는 안 된다.

## Stack Reference (for design decisions)
- **Framework**: Next.js 15 App Router, React 19
- **Styling**: Tailwind CSS 3 + shadcn/ui (New York, neutral base)
- **Icons**: Lucide React + Font Awesome (brands/solid)
- **Animation**: tailwindcss-animate, custom IntersectionObserver scroll reveal
- **Theme**: next-themes (class-based dark mode)
- **Component tokens**: `--background`, `--foreground`, `--primary`, `--border` 등 CSS variables
- **Custom tokens**: `theme` (#ffdab9 peach), `theme-sub` (#522a28 burgundy), `pale-beige`, `linen`
- **Accessibility**: WCAG AA 이상 준수 목표
